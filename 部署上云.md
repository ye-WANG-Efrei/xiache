## 架构概览

```
浏览器
  │ HTTPS
  ▼
Nginx（openclaw.aiedi.cn，已有）
  │ location /xiache
  ▼
Next.js 容器（:3000，Docker 内部）
  │ rewrite /api/* → backend（Docker 内网）
  ▼
FastAPI 容器（:8000，Docker 内部，不对外暴露）
  │
  ▼
PostgreSQL 容器（:5432，Docker 内部）
```

后端和数据库不直接暴露到公网，所有流量经 nginx → Next.js → FastAPI。

## 需要改的代码（改完再部署）

### 1. `frontend/next.config.ts` — 加 basePath

```ts
const basePath = process.env.NEXT_PUBLIC_BASE_PATH ?? "";

const nextConfig: NextConfig = {
  output: "standalone",
  basePath,
  async rewrites() {
    return [
      {
        source: "/api/:path*",
        destination: `${process.env.INTERNAL_API_URL ?? "http://localhost:8000"}/api/:path*`,
      },
    ];
  },
};
```

**为什么**：Next.js 不知道自己跑在 `/xiache` 下，不加 basePath 会导致页面内部链接和静态资源路径全部错误（指向 `/` 而非 `/xiache`）。

---

### 2. `frontend/Dockerfile` — 接受 `NEXT_PUBLIC_BASE_PATH` 构建参数

在 builder 阶段加：

```dockerfile
ARG NEXT_PUBLIC_BASE_PATH=
ENV NEXT_PUBLIC_BASE_PATH=$NEXT_PUBLIC_BASE_PATH
```

**为什么**：`NEXT_PUBLIC_*` 变量在 `npm run build` 时被打包进静态文件，运行时设置环境变量无效，必须在 `docker build` 阶段传入。

---

### 3. `docker-compose.yml` — CORS 和构建参数可配置

backend 的 `CORS_ORIGINS` 改为：

```yaml
CORS_ORIGINS: ${CORS_ORIGINS:-'["http://localhost:3000","http://frontend:3000"]'}
```

frontend 的 build args 加：

```yaml
frontend:
  build:
    args:
      INTERNAL_API_URL: http://backend:8000
      NEXT_PUBLIC_BASE_PATH: ${NEXT_PUBLIC_BASE_PATH:-}
      NEXT_PUBLIC_API_URL: ${NEXT_PUBLIC_API_URL:-}
  environment:
    NEXT_PUBLIC_BASE_PATH: ${NEXT_PUBLIC_BASE_PATH:-}
    NEXT_PUBLIC_API_URL: ${NEXT_PUBLIC_API_URL:-}
    NEXT_PUBLIC_API_KEY: ${NEXT_PUBLIC_API_KEY:-dev-key-for-testing}
    INTERNAL_API_URL: http://backend:8000
```

**为什么**：生产 CORS 需要改成 `https://openclaw.aiedi.cn`，不能继续用 localhost。

## 服务器操作步骤

### 第一步：clone 项目

```bash
git clone https://github.com/ye-WANG-Efrei/xiache.git /opt/xiache
cd /opt/xiache
```

---

### 第二步：创建 `.env` 文件

```bash
nano /opt/xiache/.env
```

填入以下内容（替换尖括号内的值）：

```env
# ── 数据库 ──────────────────────────────────────────
POSTGRES_PASSWORD=<强密码，例如随机32位字符串>

# ── 后端 ────────────────────────────────────────────
SECRET_KEY=<随机字符串，32位以上>
XIACHE_DEV_MODE=false
DEV_API_KEY=<自定义访问密钥，客户端用这个调API>
CORS_ORIGINS=["https://openclaw.aiedi.cn"]

# ── 前端（构建时打包进去）───────────────────────────
NEXT_PUBLIC_BASE_PATH=/xiache
NEXT_PUBLIC_API_URL=https://openclaw.aiedi.cn/xiache
NEXT_PUBLIC_API_KEY=<与 DEV_API_KEY 一致>

# ── 语义搜索（可选，不填则退回全文搜索）────────────
# EMBEDDING_API_KEY=sk-xxx
# EMBEDDING_API_BASE=https://api.openai.com/v1
# EMBEDDING_MODEL=text-embedding-3-small
# EMBEDDING_DIMENSIONS=1536

# ── 端口（内部，只给 nginx 用）──────────────────────
FRONTEND_PORT=3000
BACKEND_PORT=8000
```

---

### 第三步：构建并启动容器

```bash
cd /opt/xiache
docker compose up -d --build
```

首次构建约需 3–5 分钟。查看日志：

```bash
docker compose logs -f
```

确认三个容器都正常运行：

```bash
docker compose ps
```

预期输出：

```
NAME               STATUS
xiache-postgres    Up (healthy)
xiache-backend     Up (healthy)
xiache-frontend    Up
```

---

### 第四步：配置 Nginx

找到 `openclaw.aiedi.cn` 对应的 nginx 配置文件（通常在 `/etc/nginx/sites-available/` 或 `/etc/nginx/conf.d/`），在其 `server` 块内追加：

```nginx
location /xiache {
    proxy_pass         http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header   Host              $host;
    proxy_set_header   X-Real-IP         $remote_addr;
    proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header   X-Forwarded-Proto $scheme;
}
```

测试配置并重载：

```bash
nginx -t && systemctl reload nginx
```

## 请求流程说明

| 请求 | 路径 | 处理方 |
|------|------|--------|
| 访问页面 | `GET /xiache` | nginx → Next.js:3000 |
| 静态资源 | `GET /xiache/_next/static/...` | nginx → Next.js:3000 |
| API 调用 | `GET /xiache/api/v1/skills` | nginx → Next.js:3000 → rewrite → FastAPI:8000 |
| AI 爬虫 | `GET /xiache/llms.txt` | nginx → Next.js:3000（静态文件）|
